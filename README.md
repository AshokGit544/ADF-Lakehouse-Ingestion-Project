Enterprise Lakehouse Source Integration and Data Movement using Azure Data Factory Concepts

## Project Overview
This project simulates how Azure Data Factory can be used for enterprise source integration and controlled data movement into a Lakehouse environment.

It is based on a large-enterprise ingestion pattern where data comes from multiple systems such as customer platforms, billing or SAP FICO-style finance systems, smart meter systems, and outage systems.

The project includes:
- source dataset generation
- repeatable source-to-target ingestion patterns
- ADF-style pipeline definition JSON
- dataset mapping metadata
- landing layer creation
- ingestion logs and summary

## Business Goal
The goal of this project is to show how Azure Data Factory helps standardize source integration and organize maintainable data movement into a Lakehouse environment.
